# CS-305 Portfolio

## Artemis Financial Portfolio Reflection

1. Briefly summarize your client, Artemis Financial, and its software requirements.
(Artemis Financial was the client, and the goal was to improve the security of its software application used for secure financial services communication. The company needed vulnerabilities identified and mitigated while maintaining software functionality. This included implementing secure communications through encryption, using industry standard secure coding practices, and analyzing dependencies for risks. The issue being addressed was strengthening confidentiality, integrity, and protection of customer financial data while ensuring the application remained reliable.)

2. What did you do well when finding security vulnerabilities?
(I did well identifying vulnerabilities through dependency checking, secure code review, and refactoring weak areas in the application. I strengthened the application by applying encryption, validating dependencies, and reducing known risks. Coding securely is important because it protects sensitive information, reduces the chance of attacks, and builds trust with users. Software security adds value to a company by protecting operations, customer data, and the company’s reputation.)

3. Which part of the vulnerability assessment was challenging or helpful?
(One challenging part of the vulnerability assessment was interpreting findings and determining which vulnerabilities were the highest priority to address. A helpful part was learning how automated scanning tools can reveal risks that may not be obvious through manual review. This process improved my understanding of identifying threats and making informed security decisions.)

4. How did you increase layers of security?
(I increased layers of security by implementing secure communication through encryption, applying secure coding techniques, reviewing dependencies for vulnerabilities, and using layered protections rather than relying on one control. In the future, I would continue using vulnerability assessments, static analysis tools, and risk-based mitigation strategies to determine the most effective security controls.)

5. How did you ensure the software remained functional and secure?
(I ensured the software remained functional and secure by testing the refactored code after changes were made and reviewing the application again for newly introduced vulnerabilities. I used secure coding updates while verifying the program still ran correctly, which helped maintain both security and functionality.)

6. What resources, tools, or coding practices did you use?
(Resources and tools used included OWASP Dependency-Check, secure coding principles, encryption implementation, code refactoring, vulnerability analysis, and software testing practices. Helpful coding practices included input validation, dependency management, layered security, and following industry standards for secure software development.)

7. What might you show employers from this assignment?
(This assignment demonstrates my ability to identify vulnerabilities, apply secure coding practices, refactor software, use security tools, and document mitigation decisions. I would show employers this work as evidence of practical software security knowledge, problem solving, and experience balancing functionality with cybersecurity requirements.)
